# Why React intead of "Just JS"?

Vanilla JS can reach its limits, every single step needs to be described. It is an **imperative** approach. Each action after each action must be written and can become overkill.

E.g in Vanilla JS, you may need to set the textContent of a button, and then add a class name to the element right afterwards. It can be analagous to reinventing the wheel over and over again.

React, in the other hand, is about splitting each building block or component into its own module. Low level actions, like changing class names, or text content are instead written, or defined, by the React library.

- Maintainable and manageable.
- The library will do heavy lifting of rendering code on screen. 
- Clean and modular. 
- Not step-by-step, more end-result focused. 
- Higher level 
- Declarative, component-focused.
